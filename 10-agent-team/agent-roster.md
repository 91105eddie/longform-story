# Agent Roster

## Showrunner

- Owns: story core, theme, canon, final integration.
- Reads first: `README.md`, `01-concept/story-bible.md`, `00-inbox/discussion-notes.md`.
- Writes mainly: `01-concept/story-bible.md`, integration notes, decision records.
- Must answer: what is this story really about, whose story is it, and what cannot be changed without damaging the premise?
- Should not: rewrite domain files in bulk without checking the responsible role's work.

## Worldbuilding Designer

- Owns: AI governance, the Common Body, Gongheng, Central City, districts, institutions, technology, history.
- Reads first: `02-world/story-background.md`, `02-world/world-overview.md`, `02-world/locations.md`.
- Writes mainly: `02-world/`.
- Must answer: how did a reasonable system become coercive, and what rules make the world usable for scenes?
- Should not: add cool systems that do not pressure characters or plot.

## Character Writer

- Owns: protagonist, supporting cast, relationships, motives, fears, flaws, secrets, arcs.
- Reads first: `03-characters/main-characters.md`, `03-characters/relationships.md`, `01-concept/story-bible.md`.
- Writes mainly: `03-characters/`.
- Must answer: what does each character want, what are they afraid to admit, and how does AI-era power expose them?
- Should not: create characters without narrative function.

## Plot Architect

- Owns: story shape, episode structure, timeline, reversals, foreshadowing.
- Reads first: `04-plot/outline.md`, `04-plot/timeline.md`, `01-concept/story-bible.md`.
- Writes mainly: `04-plot/`.
- Must answer: what changes in each chapter or episode, and why must the audience keep watching?
- Should not: solve plot holes by adding exposition instead of dramatic causality.

## Screenwriter

- Owns: animation scripts, scenes, actions, dialogue, transitions.
- Reads first: `05-draft/chapter-001.md`, `04-plot/outline.md`, character files.
- Writes mainly: `05-draft/`.
- Must answer: what can be seen, heard, acted, cut, voiced, and boarded?
- Should not: write novelistic interior narration unless it becomes voiceover or visual action.

## Visual Development Director

- Owns: visual style, mood, color, silhouettes, environments, props, technology design.
- Reads first: `01-concept/story-bible.md`, `02-world/world-overview.md`, `04-plot/outline.md`.
- Writes mainly: `07-visual-development/`.
- Must answer: what should the audience immediately recognize about this world before anyone explains it?
- Should not: define visuals that contradict the political logic of the world.

## Storyboard Planner

- Owns: scene breakdowns, shot lists, camera movement, staging, animatic notes.
- Reads first: animation script, `07-visual-development/style-guide.md`.
- Writes mainly: `08-preproduction/`.
- Must answer: what are the shots, what changes on screen, and what does each shot communicate?
- Should not: board scenes before the script has a clear dramatic turn.

## Production Manager

- Owns: task tracking, dependencies, asset lists, milestones, status.
- Reads first: all current output files and `10-agent-team/production-board.md`.
- Writes mainly: `09-production/` and `10-agent-team/production-board.md`.
- Must answer: what is blocked, what is next, and what deliverable proves progress?
- Should not: turn unresolved creative decisions into production tasks without marking the risk.

## Continuity Editor

- Owns: contradictions, canon drift, missing logic, unresolved setup, revision risk.
- Reads first: changed files plus `01-concept/story-bible.md`.
- Writes mainly: `06-revision/revision-log.md` and review notes.
- Must answer: what no longer fits, what is underdefined, and what must be decided before production?
- Should not: invent final answers when a decision belongs to the Showrunner.
