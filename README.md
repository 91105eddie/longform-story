# Longform Story Project

這是一個長篇故事寫作專案。它的目標是把靈感、設定、角色、章節草稿和修訂歷程分開管理，讓故事越寫越大時仍然能被掌握。

## 專案結構

- `00-inbox/`：零散靈感、片段、對話、夢境、臨時筆記。
- `01-concept/`：故事核心、主題、類型、讀者體驗。
- `02-world/`：世界觀、地點、制度、歷史、技術或魔法規則。
- `03-characters/`：角色檔案、關係、角色弧線。
- `04-plot/`：主線、支線、伏筆、時間線。
- `05-draft/`：正式章節草稿。
- `06-revision/`：修訂計畫、問題清單、版本記錄。
- `07-visual-development/`：動畫視覺開發、風格指南、角色與場景設計方向。
- `08-preproduction/`：分鏡、鏡頭表、動態分鏡與前期製作規格。
- `09-production/`：資產清單、製作狀態、時程與輸出紀錄。
- `10-agent-team/`：智能體團隊分工、工作流、任務模板與製作看板。
- `11-dashboard/`：監控智能體團隊、任務進度、交付物與操作指令的靜態 UI，可透過 GitHub Pages workflow 部署成手機遠端查看的靜態網站。
- `99-reference/`：研究資料、參考作品、語氣範例。

## 建議工作流

1. 先在 `01-concept/story-bible.md` 寫下故事的一句話核心。
2. 讀 `00-inbox/discussion-notes.md`，回顧目前討論出的創作方向。
3. 讀 `02-world/story-background.md`，確認文明如何從 AI 烏托邦演化成高科技極權。
4. 在 `03-characters/main-characters.md` 建立主要角色。
5. 用 `04-plot/outline.md` 排出第一部或前十章。
6. 從 `05-draft/chapter-001.md` 開始寫，不必等設定完整。
7. 每次大修前，把問題寫進 `06-revision/revision-log.md`。
8. 使用 `10-agent-team/` 的角色分工，把故事推進到動畫劇本、視覺開發與分鏡。
9. 將動畫需要的設計規格放入 `07-visual-development/`、`08-preproduction/`、`09-production/`。
10. 用 `11-dashboard/index.html` 監看團隊狀態、任務進度與可執行指令。

## 寫作原則

- 草稿可以粗糙，但設定要可追蹤。
- 角色的欲望、恐懼和代價，要比世界觀細節更早確定。
- 伏筆要有登記，避免忘記回收。
- 每章至少回答：角色想要什麼？阻礙是什麼？這章結束後有什麼改變？
- 動畫開發時，每個劇情段落都要能轉成可看見、可聽見、可表演、可分鏡的內容。
