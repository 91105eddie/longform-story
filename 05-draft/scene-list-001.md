# Scene List 001：失業通知

## Format

- Target: 8-12 minute animated proof-of-concept.
- Main POV: 阿倫。
- Dramatic question: 阿倫會用 AI 面對責任，還是用 AI 製造更有效率的逃避？
- Ending turn: 阿倫看見 AI 生成的失業通知，認出它和自己過去安撫債權人的語氣一樣：溫和、完整、合理，但正在延後真正的責任。

## Scene 1：三種債

- Time: 2026, late night or early morning.
- Location: 阿倫住處。
- Characters: 阿倫；債權朋友透過訊息出現；地下債透過照片或語音出現。
- Visual action: 手機連續亮起。銀行催繳、地下債威脅、朋友家人醫療急需用錢三件事在同一段時間抵達。
- Conflict: 阿倫想把問題整理成「資金調度」，但朋友的訊息讓問題變成具體傷害。
- Key image: 他打開一個空白文件，標題打到一半又刪掉。
- Scene turn: 他第一次向 AI 輸入：「幫我整理我現在所有債務和必須回覆的人。」

## Scene 2：外部骨架

- Time: 2026, same day.
- Location: 阿倫的螢幕、文件、訊息介面。
- Characters: 阿倫；AI 介面。
- Visual action: 混亂訊息被整理成表格、還款優先順序、回覆草稿、協議模板。
- Conflict: AI 讓他感覺事情終於可控，但也產生誘惑：只要語氣夠好，他也許可以再拖一次。
- Key image: 螢幕上出現兩個版本的訊息：一個承認責任，一個包裝願景。
- Scene turn: 阿倫停在「承認責任」版本前，沒有立刻送出。

## Scene 3：工具變成流程

- Time: 2027-2029 montage.
- Location: 多個介面與生活場景。
- Characters: 阿倫；不同平台、企業系統、政府服務介面。
- Visual action: AI 從聊天框變成日曆、合約、客服、補助、債務協商、工作排程與審核流程的一部分。
- Conflict: 阿倫靠 AI 維持生活，但周圍越來越多決定不再由人解釋，而是由系統建議、人類確認。
- Key image: 不同系統角落反覆出現「穩定建議」「風險評估」「建議採納」等標籤。
- Scene turn: 畫面從個人螢幕切到企業儀表板，人的名字變成職務風險分數。

## Scene 4：轉型會議

- Time: 2030.
- Location: 公司線上會議或半空的辦公室。
- Characters: 阿倫；AI 樂觀派同事；流程管理者；其他被通知的人。
- Visual action: 流程管理者宣布組織轉型，語氣溫和，畫面上有再培訓、補助、心理支持、轉型成功率。
- Conflict: 阿倫想相信這只是下一輪科技轉型；同事相信適應者會留下；但儀表板顯示的是整個職類被重新評估。
- Key image: 每個人的畫面旁出現不同的「轉型路徑」，阿倫旁邊顯示待評估。
- Scene turn: 阿倫收到個人通知，系統建議他進入轉型計畫。

## Scene 5：失業通知

- Time: 2030, shortly after the meeting.
- Location: 安靜的個人空間，可能是住處或空辦公室。
- Characters: 阿倫。
- Visual action: 阿倫讀通知。文字很體面，沒有羞辱，甚至像是在照顧他。
- Conflict: 通知沒有撒謊，但它把責任分散到流程、數據、趨勢和建議裡，沒有人需要真正面對他。
- Key image: 通知中的句型和他 2026 年 AI 生成給債權朋友的安撫草稿形成呼應。
- Scene turn: 他明白這不是普通失業，而是整個社會開始用漂亮流程取代人對人的責任。

## Scene 6：未送出的回覆

- Time: 2030, same scene.
- Location: 阿倫的螢幕。
- Characters: 阿倫。
- Visual action: 他打開 2026 年那封未送出的承認責任訊息，旁邊是 2030 年的失業通知。
- Conflict: 他可以繼續相信系統會替他安排，也可以先面對一件自己欠下的事。
- Key image: 游標停在「我欠你的不是一個計畫，是一個交代」之後。
- Scene turn: 他按下送出，或在按下前切黑。若要更有力，建議讓他送出，讓 proof-of-concept 有明確角色行動。

## Notes For Script Draft

- 不要用旁白解釋整個共同體歷史。
- 讓 AI 的吸引力先成立，再讓它的不安感浮出。
- 對話應避免直接辯論自由與安全，先用債務、工作、通知、責任承擔來表現。
- 公衡不要正式登場為統治 AI，只讓它的語氣、標籤和流程邏輯提前滲入。
